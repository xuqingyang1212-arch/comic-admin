## 一、需求背景

漫剧运营后台是公司漫剧业务的内部 B 端运营平台，承担"原书 → 剧本 → 漫剧"全链路内容生产的线上化管理。当前业务上游依赖原书，编剧据原书撰写剧本，剧本经审核入库后由审核员发布制作任务，制作员领取并交付全集 / 分集成片，再经分集审核（必要时进入二审）后产出可上线的漫剧；后续若发现问题，可对漫剧发起成片修改重新进入返修版审核。整条流程涉及编剧、审核员、制作员、提审员、超级管理员等多类角色协同。

在线下或多工具拼接的旧流程下存在三类痛点：（1）剧本与制作任务流转分散在飞书 / 群消息 / 邮件，无统一查询入口，状态难追踪；（2）大体积视频（全集 5GB、分集 500MB）的上传、审核意见图文记录、版权材料归档缺乏规范化承载；（3）权限随角色变化频繁，缺乏细粒度按钮级权限矩阵和注册审核闭环，账号管理人工成本高。

本 PRD 反推后台已实现的功能现状，为后续迭代与产品交付提供基线参考文档。

## 二、需求目标

1. 实现内容生产全链路线上化管理
   覆盖原书查阅、剧本创作、剧本审核、制作任务发布、制作交付、分集 / 二审、漫剧入库、成片修改、归档下载等核心环节，统一在后台完成全部状态流转。

2. 建立标准化的状态机与审核闭环
   为剧本草稿、制作任务、审核任务、漫剧、下载任务定义固定状态枚举与单向流转规则，所有状态变更经服务端 CAS 校验，避免并发冲突。

3. 提供细粒度的角色与权限控制
   引入 5 个种子角色（超级管理员 / 编剧 / 制作员 / 审核员 / 提审员）与 49 个按钮级权限点，支持角色自建、权限矩阵编辑、邀请码注册、注册审核三步闭环。

4. 支撑大文件与海量素材的可靠上传与下载
   全集视频单文件 ≤5GB、分集 / 返修版 ≤500MB、图片 ≤10MB；支持本地存储与腾讯云 COS 双模式；下载侧通过异步 ZIP 打包 + 72 小时有效期 + Range 断点续传保障传输稳定性。

5. 强化操作安全与多端协同
   全局接入单设备登录（同一账号最多 1 端在线）、3 秒重复提交防护、JWT 24 小时有效期、所有写操作的日志记录与审核意见留底。

## 三、名词解释

| 名词 | 说明 |
|------|------|
| 原书 | 漫剧创作的上游内容资产（小说原文），存于"书籍管理"，含分类、标签、付费卡点 |
| 剧本草稿 | 编剧创作中、未通过审核的剧本，含 5 种状态（待提审 / 待认领 / 审核中 / 驳回修改 / 审核不通过） |
| 剧本 | 已审核通过、入库的成品剧本，作为发布制作任务的输入 |
| 剧本二创 | 基于已入库剧本派生新草稿（剧本类型设为"多版本"，关联 `originalScriptId`） |
| 制作任务 | 由审核员基于剧本发布的漫剧制作工单，分"制作"与"修改"两类 |
| 全集 | 制作的第一阶段交付：单文件包含所有集数的完整视频（≤5GB） |
| 分集 | 制作的第二阶段交付：每集 1 个有字幕视频 + 1 个无字幕视频（单文件 ≤500MB） |
| 返修版 | 漫剧上线后通过"成片修改"产生的修订版分集交付物 |
| 二审 | 当分集审核员被配置了"上级审核员"时，分集审核通过后进入的二次审核环节 |
| 审核任务 | 与制作任务 1:1 关联的审核工单，按阶段分为全集审核 / 分集审核 / 返修版审核 / 二审审核 |
| 审核意见 | 审核任务下的反馈记录，每条含文字 + ≤15 张图片，按顺序保存 |
| 漫剧 | 制作完成（分集审核或二审通过）后入库的可上线视频内容 |
| 成片修改 | 在漫剧管理对已上线漫剧发起的修订流程，会产生新的"修改"类制作任务 |
| 下载任务 | 异步打包某漫剧的有字幕视频 / 无字幕视频 / 提审材料为 ZIP 的工单，链接 72 小时有效 |
| 邀请码 | 由角色管理生成的 HMAC 签名字符串，注册者凭此码绑定指定角色提交注册申请 |
| 注册审核 | 注册申请的人工审核环节，通过后才会创建实际用户账号并绑定角色 |
| 二审用户 | 在用户管理中为审核员配置的上级审核员，分集审核通过后由其继续二审 |

## 四、业务流程

### 4.1 内容生产主链路

1. 编剧基于原书在"剧本创作"创建剧本草稿
2. 编剧提交审核
   - 草稿未指定审核员：状态变为"待认领"，进入剧本审核任务大厅
   - 草稿已指定审核员：状态变为"审核中"，直接归属该审核员
3. 审核员处理草稿
   - 审核通过：草稿置为"审核通过"，同时按草稿信息复制创建剧本（入库到"剧本管理"）
   - 驳回修改：草稿置为"驳回修改"，编剧可继续修改后重新提交（回到步骤 2）
   - 审核不通过：草稿置为"审核不通过"，终态，不可继续操作
4. 审核员在"剧本管理"对剧本发布制作任务
   - 填写画风类型 / 视觉效果 / 画面比例 / 制作备注
   - 系统创建一条 `制作` 类型的制作任务，状态"待认领"，发布到"任务大厅"，发起人 = 审核员 = 当前用户
5. 制作员在"任务大厅"领取制作任务
   - 领取后任务状态变为"全集制作中"，归属当前制作员
6. 制作员在"我的任务"上传全集视频
   - 提交后任务状态变为"全集审核中"，同时创建 / 更新审核任务（任务类型 = 全集审核，状态 = 审核中）
7. 审核员在"漫剧审核 - 待我审核"处理全集审核
   - 审核通过：审核任务状态变为"待提审"（等待分集），制作任务状态变为"分集制作中"
   - 驳回修改：审核任务状态变为"驳回修改"，制作任务回退至"全集制作中"
8. 制作员上传分集（每集 1 个有字幕 + 1 个无字幕视频）
   - 提交后审核任务复用同一条记录，类型变为分集审核，状态变为"审核中"，制作任务变为"分集审核中"
9. 审核员处理分集审核
   - 审核通过：
     - 若审核员未配置二审用户：制作任务变为"已完成"，按制作任务 + 交付物自动创建漫剧入库
     - 若审核员配置了二审用户：审核任务的 `task_type` 切换为"二审审核"、归属新审核员，制作任务变为"二审审核中"
   - 驳回修改：审核任务状态变为"驳回修改"，制作任务回退至"分集制作中"
10. 二审审核员处理二审任务（仅二审场景）
    - 审核通过：制作任务变为"已完成"，按制作任务 + 交付物自动创建漫剧入库
    - 驳回修改：审核任务状态变为"驳回修改"，制作任务回退至"分集制作中"

### 4.2 成片修改子链路

1. 审核员在"漫剧管理"对已入库漫剧发起成片修改，填写修改意见
2. 系统创建一条 `修改` 类型的制作任务，状态"返修版制作中"；同时创建审核日志保存意见快照；按原漫剧封面 / 版权图 / 集数文件预填一条返修版交付草稿
3. 制作员在"我的任务"上传返修版视频
4. 提交返修版审核：审核任务复用同一条记录，类型变为"返修版审核"，状态"审核中"；首次提交时从意见快照初始化审核意见
5. 审核员处理返修版审核
   - 审核通过：制作任务变为"已完成"，按返修版交付物更新原漫剧的封面 / 集数文件
   - 驳回修改：制作任务回退至"返修版制作中"

### 4.3 注册与登录子链路

1. 角色管理员在"角色管理"为指定角色复制邀请链接（邀请码 = HMAC-SHA256 签名后的角色 ID）
2. 注册者打开邀请链接，进入注册页
3. 注册者填写邮箱 / 用户名 / 密码 / 验证码（当前固定 000000）+ 提交
4. 系统校验：邮箱格式、密码 6~24 位、用户名 1~32 位、邮箱未被注册（含审核中申请去重）
5. 校验通过后写入 `registration_requests` 表，状态"审核中"，弹出"注册成功，请等待管理员审核"提示
6. 系统设置 - 注册审核中处理申请
   - 通过：事务内创建用户 + 绑定角色，申请置"审核通过"
   - 不通过：申请置"审核不通过"
7. 注册者凭邮箱密码登录
   - 用户存在 + 密码正确 + 状态 = 启用：生成新 session_token + JWT，登录成功
   - 用户不存在但有"审核中"申请：返回"账号正在审核中"
   - 用户存在但状态 = 禁用：返回"账号已禁用"

### 4.4 下载子链路

1. 用户在"漫剧管理"或"漫剧详情"点击下载，选择内容类型（有字幕视频 / 无字幕视频 / 提审材料）
2. 系统检查同漫剧 + 同内容类型是否已有未过期的"已完成"下载任务
   - 存在且未指定 force：返回 `duplicate=true`，前端弹出二次确认"下载中心已存在该下载任务，是否重新下载？"
   - 用户确认重下：标记旧任务为"已失效"，创建新任务
   - 不存在：直接创建新任务，状态"进行中"
3. 后端异步打包：按内容类型遍历漫剧分集 / 封面 / 版权图 → 创建 ZIP（Store 模式不压缩，支持 ZIP64） → 写入 `uploads/downloads/`
4. 打包成功：任务状态置"已完成"，写入 `file_url`，`expires_at = now + 72h`；失败置"已失败"
5. 用户在"下载中心"点击下载：服务端校验状态 + 过期时间 → 返回带 host 的下载 URL（支持 Range 断点续传）
6. 失败任务可点击"重试"：状态回到"进行中"，重新走打包流程

## 六、全局说明

> 以下条目直接引用项目全局规范，详见 `global-spec.md` 同名段落。

### 6.1 筛选项（按 §G1）
- 流式布局，自动换行；查询 / 重置按钮右对齐
- 文本框提交时 `.trim()`；ID 类精确匹配，名称类模糊匹配
- 查询 / 重置时分页页码重置为 1

### 6.2 表头（按 §G2）
- 表头 `sticky top-0 z-10`，纵向滚动时常驻
- 不允许合并单元格

### 6.3 分页（按 §G3）
- 默认每页 10 条；可选 10 / 20 / 50 / 100；后端硬上限 100
- 分页固定在卡片底部，左侧"共 N 条记录"，右侧上一页 / 页码 / 下一页 / 每页选择
- 切换 pageSize 时自动 `safePage = min(currentPage, totalPages)`

### 6.4 数据展示（按 §G4）
- 单元格 `whitespace-nowrap`，禁止换行
- 列数较多时表格 `overflow-x-auto`
- 操作列以 `gap-3` 间隔横向排列文字链接

### 6.5 字数统计（按 §G5）
- 计数规则：去除全部空白后字符长度
- 试读字数 = 全文字数 × 5%

### 6.6 文件上传（按 §G8）
- 支持格式：视频 MP4 / MOV，图片 JPG / JPEG / PNG / GIF / WebP
- 容量上限：全集视频 5GB，分集 / 返修版 500MB，图片 10MB，版权 ≤15 张/漫剧，审核意见 ≤15 张/条
- 上传链路：前端 presign → COS 直传或本地中转 → 返回 `/uploads/<key>`

### 6.7 视频播放器（按 §G9）
- 内嵌 HTML5 `<video>` 自定义控件，支持播放 / 暂停 / 进度拖拽 / 全屏 / 倍速

### 6.8 图片显示（按 §G10）
- 缩略图 52×52px，点击弹出预览弹窗
- 多图模式支持左右切换、键盘 ←/→、Esc 关闭

### 6.9 重复提交防护（按 §G11）
- 全局拦截：同 user + method + path + bodyHash 在 3 秒内重复 POST/PUT/DELETE
- 跳过 `/upload/`、`/save`、`/draft`
- 命中返回 HTTP 429 + "请勿重复提交"

### 6.10 二次确认（按 §G13）
- 取消任务、驳回修改、审核不通过、删除草稿、发起成片修改、注册不通过 必须弹窗确认

### 6.11 通知机制（按 §G15）
- 当前未对接任何外部通知（飞书 / 邮件 / 短信 / 站内 / Webhook）
- 设备被踢仅在用户下次操作时返回 HTTP 409 文案

### 6.12 登录与会话（按 §G16）
- 邮箱密码登录，JWT (HS256) 默认 24 小时
- 单设备登录：登录时刷新 `session_token`，旧 session 立即失效

### 6.13 路由与权限拦截（按 §G17）
- 前端 SPA 式 selectedKey + URL hash 持久化（独立路由仅登录 / 注册 / 忘记密码）
- 启动时拉取 `/users/me` 写入权限上下文，菜单 / 按钮按权限过滤
- 后端：JWTAuth → SessionGuard → LoadPermissions → PreventDuplicateSubmit → RequirePerm

## 七、功能详情

### 7.1 资源管理 - 书籍管理

#### 7.1.1 业务流程说明

1. 编剧或审核员在「资源管理 → 书籍管理」浏览原书清单
2. 点击书名进入书籍详情抽屉，查阅全文与付费卡点
3. 点击「创作剧本」打开剧本编辑器抽屉，新建一条剧本草稿（保存即落库）
4. 提交后转入剧本创作 / 剧本审核流程

#### 7.1.2 状态流转表

> 书籍本身无状态机，仅根据「上架时间」「关联剧本数」呈现。

#### 7.1.3 角色权限矩阵

| 菜单 | 按钮 | 超级管理员 | 编剧 | 制作员 | 审核员 | 提审员 |
|------|------|-----------|------|--------|--------|--------|
| 书籍管理 | 列表数据 | ● | ● | - | - | - |
|         | 书籍详情 | ● | ● | - | - | - |
|         | 创作剧本 | ● | ● | - | - | - |

#### 7.1.4 页面清单

| 页面 | 介绍 |
|------|------|
| 书籍管理列表 | 见 7.1.4.1 |
| 书籍详情抽屉 | 见 7.1.4.2 |
| 创作剧本抽屉 | 见 7.1.4.3 |

##### 7.1.4.1 书籍管理列表

**入口**
1. 侧边栏「资源管理 → 书籍管理」

**筛选项**
- 书籍ID：文本框，非必填，精准搜索；提交时 trim 前后空格
- 书籍名称：文本框，非必填，模糊搜索
- 内容类型：单选下拉，非必填，可选项「原作」「多版本」
- 是否关联剧本：单选下拉，非必填，可选项「是」「否」，默认「否」

**字段说明**

  - 返修版：制作任务 → 已完成；按返修版交付物更新原漫剧
- 驳回修改：调用 `.../review {result:"驳回修改", opinions}`；意见必填；制作任务回退到对应"制作中"状态；审核任务 → 驳回修改

## 八、异常场景

| 异常场景 | 触发条件 | 系统行为 | 用户反馈 |
|---------|---------|---------|---------|
| 网络中断 | 提交时网络断开 | 不修改服务端状态；前端 fetch 抛出错误 | toast：网络异常或后端文案，请重试 |
| JWT 过期 | 24 小时后访问 | 后端返回 HTTP 401 + "令牌无效或已过期" | 前端清 token，跳转 `/login` |
| 单设备被踢 | 同账号在新设备登录 | 后端 SessionGuard 返回 HTTP 409 | 弹窗 "您的账号已在其他设备登录，当前会话已失效" |
| 重复提交 | 同 user/method/path/bodyHash 在 3 秒内重复 POST/PUT/DELETE | HTTP 429 拦截 | toast "请勿重复提交" |
| 全集视频超大 | 上传文件 > 5GB | 前端在选择文件后即拦截（`<input>` accept 不限制大小，需补强）`[?]` | 上传前提示文件大小超限 |
| 分集 / 返修版视频超大 | 单文件 > 500MB | 同上 | 同上 |
| 图片超大 | 单张图片 > 10MB | 同上 | 同上 |
| 上传文件类型错误 | 选择非 MP4 / MOV 视频 或非 JPG/PNG/GIF/WebP 图片 | `<input accept>` 自动过滤；后端不再二次校验 `[?]` | 文件选择器仅显示符合类型的文件 |
| 并发领取（剧本 / 制作任务） | 两人同时点击"领取" | CAS 校验 RowsAffected = 0 | toast "该任务已被领取" |
| 并发审核 | 两人同时点击"审核通过 / 驳回" | CAS 校验 RowsAffected = 0 | toast "该任务已被处理，请勿重复操作" |
| 并发取消 | 状态已变更后再点取消 | 同上 | toast "该任务已被处理，请勿重复操作" |
| 状态非法（编辑已通过的草稿） | 草稿状态 ∉ {待提审, 驳回修改} 时调用 PUT | HTTP 400 | toast "当前状态不可编辑" |
| 状态非法（领取非待认领任务） | 任务状态 ≠ 待认领 时调用 claim | HTTP 400 | toast "当前状态不可领取" |
| 状态非法（取消已完成 / 已取消） | 任务状态 ∈ {已完成, 已取消} 时调用 cancel | HTTP 400 | toast "当前状态不可取消" |
| 状态非法（删除非待提审） | 草稿状态 ≠ 待提审 时调用 DELETE | HTTP 400 | toast "仅待提审状态可删除" |
| 重复邮箱注册 | 提交注册申请时邮箱已在 users 或审核中申请 | HTTP 400 | toast "该邮箱已注册" / "该邮箱已提交注册申请，请等待审核" |
| 重名剧本入库 | 剧本审核通过时剧本名已在 `scripts` 表 | HTTP 400 | toast "剧本名称「XXX」在剧本库中已存在，请修改后再审核通过" |
| 重名漫剧入库 | 漫剧审核通过时剧集名已在 `comics` 表 | HTTP 400 | toast "剧集名称「XXX」在漫剧列表中已存在，请修改后再审核通过" |
| 重复修改任务 | 漫剧已有进行中的修改任务时再次发起修改 | HTTP 400 | toast "该漫剧已有进行中的修改任务" |
| 权限不足（API） | 缺少接口权限 | HTTP 403 | toast 后端返回的"无权限"文案 |
| 权限被中途取消 | 角色 / 权限调整后访问无权页 | 前端 `selectedKey` 重定向到首个有权限页 | 静默重定向 |
| 下载链接已过期 | 链接超过 72h 后访问 | HTTP 400，状态置"已失效" | toast "下载链接已过期" |
| 下载任务打包失败 | 文件缺失 / 写盘失败 | 状态置"已失败" | 列表显示失败状态，可点击"重试" |
| 下载二次创建（同类型已完成） | force=false 且已存在未过期完成任务 | 接口返回 `duplicate=true` | 弹"下载中心已存在该下载任务，是否重新下载？" |
| 上游接口失败 / 数据缺失 | 详情接口找不到 ID | HTTP 404 | toast 后端返回的 "XXX 不存在" |
| 注册验证码错误 | 非 `000000` | HTTP 400 | toast "验证码错误" |
| 注册密码格式错误 | 不符合 6~24 位规则 | HTTP 400 | toast "密码需 6~24 位..." |
| 注册用户名超长 / 过短 | 长度 < 1 或 > 32 | HTTP 400 | toast "用户名长度需在 1~32 位" |
| 邮箱格式错误 | 不符合正则 | HTTP 400 | toast "邮箱格式不正确" |
| 审核驳回意见缺失 | 漫剧驳回时无任何文字 / 图片 | HTTP 400 | toast "驳回时审核意见必填" |
| 注册审核非"审核中"状态被处理 | 重复处理已通过 / 不通过的申请 | HTTP 400 | toast "该申请不在审核中状态" |
